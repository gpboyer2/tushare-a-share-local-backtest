# 安全说明

## 1. 安全边界

这个项目会涉及以下敏感对象：

- Tushare Token
- 本地 SQLite 数据库
- 回测输出结果
- 浏览器工作台里的 SQL 执行能力

请默认按以下规则处理：

- 不要在代码、文档、Issue、PR 中提交真实 Token
- 不要把本地数据库、日志、回测输出直接公开
- 不要把包含真实业务数据的截图直接放到公开页面

## 2. 漏洞或安全问题怎么报告

如果你发现下面这类问题，请不要直接公开发 Issue：

- Token 泄露
- 任意文件读取或写入
- 任意命令执行
- 数据库管理接口越权
- 敏感配置泄露

建议处理方式：

1. 先准备复现步骤
2. 说明影响范围
3. 说明你确认过的最小触发条件
4. 私下联系维护者处理

在修复发布前，不建议公开披露利用细节。

## 3. 当前项目特别需要注意的点

### Tushare Token

- 只能通过环境变量传入
- 不要写死在配置文件、脚本或提交记录里

### SQLite 数据文件

- `data_cache.db` 是主库
- `*.db-wal` 和 `*.db-shm` 是 SQLite 的伴生文件
- 这些都不应该提交到 Git 仓库

### 数据库管理能力

项目带有数据库浏览和写入功能。

如果你把这个项目部署成长期可访问服务，必须额外加上：

- 认证
- 权限控制
- 审计日志
- 请求限流

当前仓库默认是本地研究工具，不是面向公网的生产级服务。

## 4. 免责声明

本项目默认用于本地研究、策略验证和工程学习。

请不要把当前仓库直接当成生产交易系统或公网数据库服务使用。

## 5. 数据库管理模块的公开边界

当前仓库内置了数据库浏览、SQL 执行和行级写入能力。

这套能力默认只适合：

- 本机开发
- 本机研究
- 本机调试

不适合直接暴露到公网或开放给不可信用户。

如果你要对外提供服务，至少要自行补齐：

- 身份认证
- 权限控制
- 请求限流
- 审计日志
- 风险隔离
